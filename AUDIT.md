Аудит шаблонов проекта: соответствие стандартам коммерческого магазина
Я изучил все 38 HTML-шаблонов проекта. Ниже — систематизированный отчёт по категориям.

🔴 Критические проблемы (блокируют коммерческую эксплуатацию)
1. Формы авторизации без стилизации
login.html:10 и registration.html:10

Используется {{ form.as_p }} — рендерит «голые» поля без Bootstrap-классов. В коммерческом магазине формы входа/регистрации — точка входа, они должны выглядеть профессионально.

2. Нет детализации заказа
views.py:302-303 — UserOrderDetailView — пустой pass. Пользователь не может увидеть состав своего заказа. Это базовая функция любого магазина.

3. Страница успешного заказа без состава
checkout_success.html — нет списка купленных товаров. Клиент не видит, что именно он заказал.

4. Футер с заглушками
_footer.html:20-31 — «Доставка скоро появится», «VK скоро появится» и т.д. Для коммерческого магазина это выглядит как незаконченный сайт, убивает доверие.

5. Нет юридических страниц
Отсутствуют: Политика конфиденциальности, Пользовательское соглашение, Условия возврата, Оферта. По ФЗ-152 и правилам маркетплейсов — обязательно.

6. Нет баннера согласия на cookies
Требуется ФЗ-152 (и GDPR при иностранном трафике).

🟠 Серьёзные проблемы (снижают конверсию и SEO)
7. Нет хлебных крошек (breadcrumbs)
Ни на одной странице. Для SEO (микроразметка BreadcrumbList) и навигации — обязательно в e-commerce.

8. Нет микроразметки Schema.org (JSON-LD)
Отсутствует разметка Product, Offer, BreadcrumbList, Organization. Без неё — нет Rich Results в Google, нет интеграции с Google Shopping.

9. Нет Open Graph мета-тегов
_head.html — нет OG-тегов. При шаринге в соцсетях — некрасивый превью.

10. Нет канонических URL
Отсутствует <link rel="canonical">. Приводит к дублям в индексе поисковиков.

11. Каталог без фильтров и сортировки
product_list.html — нет сортировки (по цене, новизне), нет фильтров (по размеру, цвету, цене). Стандарт для любого магазина.

12. Нет похожих/рекомендуемых товаров на карточке товара
product_details.html — нет блока «С этим товаром покупают» / «Похожие товары». Упущенная кросс-селл конверсия.

13. Нет галереи/карусели изображений товара
product_details.html:23-33 — миниатюры не кликабельны, нет зума. В коммерческих магазинах — обязательна интерактивная галерея.

14. Нет индикатора остатков
Нет сообщений типа «Осталось 3 шт.» — важнейший драйвер срочности покупки.

15. Нет информации о доставке на карточке товара
Клиент не видит условий доставки до добавления в корзину. Это снижает конверсию.

16. Нет быстрой кнопки «В корзину» в карточке товара каталога
_product_card.html:46 — только «Подробнее». Коммерческие магазины дают quick-add.

17. Нет мини-корзины в шапке
_header.html:26-36 — только ссылка на страницу корзины. Dropdown с содержимым — стандарт.

🟡 Проблемы качества кода
18. Дублирование JS-функций
showNotification, getCookie, updateCartCounter — полностью дублируются в product_details.html и cart.html. Нужно вынести в общий JS-файл.

19. Скрипт вне блока extra_scripts
product_details.html:82-177 — <script> размещён прямо внутри {% block content %}, а не в {% block extra_scripts %}. Это нарушает архитектуру и замедляет рендер.

20. console.log в продакшн-коде
product_details.html:83-87 — отладочные логи оставлены.

21. Дублирование CSRF-токена в корзине
cart.html:6-7 — и {% csrf_token %}, и <input type="hidden" name="csrfmiddlewaretoken">. Первый уже генерирует второй.

22. Мёртвый блок в _scripts.html
_scripts.html:4 — {% block extra_scripts %} внутри {% include %} не работает (блоки не пробрасываются через include). Дублирует блок из base.html.

23. Форма профиля без Bootstrap-виджетов
profile_form.html:20-29 — {{ field }} без form-control классов. Поля будут нестилизованными.

🔵 Мелкие недочёты
Нет favicon для мобильных — только favicon.ico, нет apple-touch-icon, android-chrome-192x192.png и т.д.
Нет <meta name="robots" content="noindex"> на страницах корзины, чекаута, профиля — они не должны индексироваться.
Нет кастомных 404/500 страниц — важны для удержания пользователя.
Нет перепрослушки подтверждения email — если письмо не дошло, нет кнопки «Отправить снова».
Нет страницы «Избранное»/wishlist — стандартная функция e-commerce.
Нет отзывов/рейтингов товаров — важный социальный proof.
Нет кнопки «Сообщить о наличии» для товаров not in stock.
